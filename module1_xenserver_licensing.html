= Entitlement 제공

Citrix for Private Cloud Universal Hybrid Multi-Cloud Citrix Platform License
Per-Socket 모델은 매우 단순합니다. 서버의 물리 CPU 소켓 수만 세면 됩니다. 코어 수는 관계없습니다. VMware가 코어 단위로 전환한 것과 대비하면, 이 단순함이 XenServer의 큰 장점입니다.

  빈 소켓(Unpopulated Socket)은 라이선스 계산에서 제외됩니다. 풀 내 모든 호스트가 적절한 라이선스를 보유해야 '풀 라이선스됨' 상태가 됩니다.
라이선스 산정의 핵심은 '채워진 소켓'만 세면 된다는 것입니다. 코어 수는 관계없습니다. 이것이 VMware Per-Core 모델 대비 큰 장점입니다.

  공식 문서 상 '무료 DR 라이선스'에 대한 명시적 언급이 없으므로, 정확한 DR 정책은 반드시 Citrix 파트너/영업 담당자와 사전에 확인해야 합니다.
DR 라이선스는 고객이 자주 혼란스러워하는 부분입니다. Cold Standby는 미할당 가능하지만 가동 시 재할당이 필요합니다. Active-Active는 양쪽 모두 Full License가 필수입니다. 무료 DR 라이선스에 대한 명시적 정책이 없으므로 반드시 영업 채널을 통해 확인하세요.

Checklist & Guidelines

다중 사이트 설계 체크포인트

01
동일 풀 라이선스 일관성

풀 내 모든 호스트는 동일 에디션의 라이선스를 적용해야 합니다. 부분 적용 시 풀 전체 기능 제한이 발생합니다.

02
VM 이동성 라이선스 적합성

HA 또는 XenMotion을 통한 라이브 마이그레이션 시, 대상 호스트에도 유효한 라이선스가 있어야 합니다.

03
라이선스 서버 가용성 확보

다중 사이트 시 이중화 구성 또는 각 사이트에서 접근 가능한 네트워크 경로를 확보해야 합니다.

04
DR Runbook 반영

DR 가동 시나리오에 라이선스 재할당 및 적용 프로세스를 명시적으로 포함해야 합니다.

Architect's Note
"기술적인 DR 구성만큼 중요한 것이 라이선스 컴플라이언스입니다. 긴급 상황에서 라이선스 만료나 부족으로 VM이 시작되지 않는 최악의 상황을 방지하기 위해, 정기적인 라이선스 모의 훈련이 필요합니다."

  Grace Period(유예 기간) 30일 — 라이선스 서버 연결 불가 시 30일간 기능 유지, DR 초기 대응 시간 확보에 활용

이 네 가지 체크포인트는 다중 사이트 설계 시 반드시 확인해야 합니다. 특히 DR Runbook에 라이선스 관련 절차를 포함하는 것을 잊지 마세요.

Risk Mitigation — 선제적 장애 예방

호스트 추가나 CPU 소켓 구성 변경 작업 전, 라이선스 여유분을 반드시 먼저 확인하는 절차를 변경 관리 프로세스(Change Management)에 포함시키십시오.

  Operator's Guide
"라이선스 장애는 예고 없이 서비스 중단을 초래할 수 있는 크리티컬한 운영 리스크입니다. 단순한 수량 관리를 넘어, 인증서와 서버 상태까지 포함된 포괄적인 모니터링 체계가 필요합니다."

유용한 CLI 명령어

# 라이선스 서버 정보 확인
xe pool-param-get uuid=<pool-uuid> param-name=license-server
# 호스트 라이선스 상태 조회
xe host-license-view

  주기적인 스크립트 실행으로 상태를 로깅하는 것을 권장합니다.

운영팀이 반드시 알아야 할 CLI 명령어 두 가지를 소개합니다. 주기적으로 이 명령어를 스크립트로 실행하여 라이선스 상태를 모니터링하세요.

핵심 요약 & Next Step

핵심 메시지 (Key Takeaways)

별도 라이선스 필수

CVAD/DaaS 라이선스만으로는 XenServer 8.4 권한이 없습니다. Premium Edition 소켓 라이선스를 반드시 별도로 확보해야 합니다.

온프레미스 서버 필수

Citrix Cloud 기반 라이선싱은 아직 지원되지 않습니다. 온프레미스 Citrix License Server 구축이 필수 요구사항입니다.

플랫폼 구독 최적화

개별 구매보다 플랫폼 구독(Entitlement)을 활용하면 관리 복잡도를 줄이고 비용 효율성을 높일 수 있습니다.
