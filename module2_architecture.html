Section 2-1 · 15분

Resource Pool & Coordinator 구조

Resource Pool 개념

  • 여러 호스트를 하나의 단일 엔터티로 관리
  • 단일 인증 지점 — 개별 로그인 불필요
  • 공통 네트워크/스토리지 프레임워크 공유
  • VM 마이그레이션, 워크로드 밸런싱
Dom0를 문서화되지 않은 방식으로 변경 시 Upgrade 문제 발생 가능

Coordinator 역할

Pool 내 Host 중 1대만 Coordinator로 선출

  • Pool 전체 Single Control Point
  • Configuration Data 유지
  • Distributed Locking 처리
  • Member Server 구성 관리

장애 시 자동 복구

  • Config Data는 각 Host 로컬에 복제
  • Coordinator 장애 → 다른 Host가 역할 승계
  • 기존 실행 중 VM은 영향 없음
  • 관리 작업(시작/중지/이동)만 불가
핵심: Coordinator 없어도 기존 VM은 계속 동작
Coordinator가 죽어도 기존 VM은 계속 돌아갑니다. 당황하지 마세요. 다만 새로운 VM을 시작하거나 설정을 변경하는 '관리 작업'은 불가능합니다.

Section 2-2 · 15분

HA (High Availability) 동작 원리

Network Heartbeat

  • UDP 694로 Pool 내 모든 호스트에 전송
  • 고정 IP 필요
  • 분리된 bonded 관리 네트워크 필요

Storage Heartbeat

  • 공유 스토리지 (iSCSI/NFS/FC/GFS2)
  • 4GB LUN 공간에 상태 기록
  • 60초 타임아웃으로 Liveset 유지

장애 발생 시 동작

상황동작
Network Heartbeat 일부 손실 + Storage OKLargest Partition 판단 → 아니면 Self-Fence
Storage 실패 + Network OK전체 Host 동일하면 계속, 일부 불가 → Self-Fence
Network + Storage 모두 손실즉시 Self-Fence
Toolstack(XAPI) 비정상Self-Fence
인프라가 준비되지 않은 상태에서 HA를 켜면, 보호가 아니라 더 큰 장애를 유발합니다. 현장에서 가장 많이 보는 실수가, 인프라 준비 없이 HA를 켜놓고 더 큰 장애를 겪는 케이스입니다.

HA vs Live Migration 판단 기준

다음 조건 중 2개 이상 해당되면 HA보다 Live Migration이 더 안전

#운영 환경 조건기술적 위험
1호스트 2대 구성Split-brain 중재 어려움
2관리 네트워크 단일 구성오탐지로 정상 VM 재기동 위험
3Shared Storage 불안정Heartbeat Disk 실패 → HA 오동작
4VM 수가 적은 환경오동작 리스크 > 자동화 이점
5재기동에 민감한 서비스HA는 VM 재부팅 동반
6운영자 상시 대응 가능수동 판단이 더 안전
7계획 점검 위주 환경무중단 이동이 운영 핵심
  Key Takeaway: "자동화보다 통제가 중요한 환경에서는 Live Migration이 더 안전합니다"
이 판단표는 핸드아웃으로 배포합니다. 현장에서 고객과 함께 7개 항목을 체크하면 HA 적용 여부를 합리적으로 결정할 수 있습니다.

HA 안전 구성 체크리스트

5개 카테고리, 모든 항목이 YES여야 안전

1
네트워크 구성Bond 구성, 이중 스위치, 관리망/VM 트래픽 분리
2
스토리지/HA SRShared Storage, 다중 경로, Failover 테스트 완료
3
용량 설계 (N-1)1대 장애 시 남은 Host가 모든 VM 수용 가능
4
VM 정책Protected VM 최소화, Restart Priority 구분, Anti-Affinity
5
운영 정책수동 개입 금지 정책, 로그 선 확인 절차
  "하나라도 NO가 있다면, HA는 보호 장치가 아니라 시스템 전체 장애를 유발할 수 있는 위험 요소"
이 체크리스트는 핸드아웃으로 배포합니다. 고객 사이트에서 HA 구성 전에 반드시 이 5가지를 확인하세요.

모듈 2 Q&A

Q. Coordinator가 죽으면 VM도 죽나요?
A. 아닙니다. 기존 실행 중 VM은 영향 없음. 관리 작업만 불가합니다.

Q. NFS와 iSCSI 중 어떤 것이 좋나요?
A. DR 필요 → LVM(iSCSI/FC), Thin Provisioning 필요 → NFS 또는 GFS2

Q. Pool에 최대 몇 대 Host를 넣을 수 있나요?
A. LVM/NFS: 32대, GFS2: 16대
